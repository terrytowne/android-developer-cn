安卓风格

上一节 下一节

大多数开发者希望将他们的应用分发到不同的平台上。如果你打算为安卓开发应用，请记住在不同的平台需要遵守不同的要求和惯例。在某个平台上开起来不错的设计，也许在另一个平台上就不合适。“一次设计，到处运行”的想法可能在一开始能节省一些时间，但是和平台不一致的体验最终可能会疏远用户。考虑按照下面的指导进行设计，避免常见的错误和缺陷。

 

不要模仿其它平台的用户界面元素

不同的平台都会提供别具一格的、精心设计的、主题化的界面元素。例如在一些平台上鼓励使用圆角按钮，另一些则鼓励使用渐变标题栏。许多情况下，虽然元素的功能是一样的，但是设计方法截然不同。

当你为安卓设计应用时，不要使用其它平台的用户界面元素，也不要模仿其它平台元素的行为。查看设计指南的构造块部分来了解安卓里最重要的用户界面元素和它们在系统默认主题下的样子。同时，观察安卓系统应用，了解这些元素是如何使用的。如果你要自定义用户界面元素，请按照你品牌的统一设计，不要照搬其它平台的设计。

Android、iOS 和 Windows Phone 的 UI 元素样例。
 

不要使用专为其它平台设计的图标

不同的平台都会提供常用功能的图标集，例如分享、新建和删除。

如果你正在将应用移植到安卓平台，请不要使用专为其它平台设计的图标。

你可以在下载页面找到各种图标供你使用。

Android、iOS 和 Windows Phone 的图标样例。
 

不要使用底部标签栏

其它平台使用底部标签栏在应用中切换视图。按照平台管理，安卓选项卡视图空间显示在屏幕顶端的操作栏中。不仅如此，安卓应用可以使用底部操作栏来显示操作。

你应当按照该指导设计应用，提供统一的平台应用体验，区分操作栏和视图切换。

更多关于如何正确使用操作栏来控制视图，参见操作栏指南。

Android 拨号应用的操作栏标签 vs. iOS 的底部标签栏。

不要通过硬编码连接其它应用

有时你希望你的应用可以利用其它应用中的功能。例如通过社交网络或者短消息分享应用中的内容，或者在浏览器内打开链接。不要硬编码这些操作，否则就只能连接到一个特定的应用。你应该使用安卓提供的意图 API 打开一个活动选择器，它将会列出所有支持指定操作的应用。这样用户可以选择他们喜欢的应用完成操作。关于分享操作，你可以在操作栏中使用分享操作提供器来提供更快的访问用户最近使用分享目标的方法。

使用活动选择器或者操作栏中的分享操作提供器来连接到其它应用。
 

不要在操作栏中使用带有标题的返回按钮

其它平台使用带有标题的返回按钮，使用户可以返回应用的上一层。安卓则使用操作栏的应用图标返回上一层，同时使用导航栏的返回按钮返回前一个屏幕。更多相关信息，请阅读导航模式。

按照该指导设计应用，提供统一的导航体验。

Android 操作栏中的向上图标 vs. iOS 的返回按钮。

不要在列表中使用向右箭头

在其它平台上通常在列表中使用向右箭头提示用户触摸后有更多的内容。

安卓中在下级菜单项上不使用这样的指示箭头。和平台惯例保持一致，这样用户就不会猜测这些箭头表示什么意思。

Android 设置列表中没有向右箭头 vs. iOS 设置。

设备无关性

记住你的应用会运行在各种不同屏幕尺寸的设备上。为不同的屏幕尺寸和密度创造不同的图片文件，使用多面板布局这样的方法来在不同的设备因素里适当调整你的用户界面。

更多相关信息，阅读设备与显示以及多面板布局设计指南。
